# 产品主页设计调研

## 调研对象

| 产品 | 类型 | 配色 | 核心特点 |
|------|------|------|---------|
| **Raycast** | macOS 启动器 | 深色+紫色 | 键盘动画、极简标语、扩展生态网格 |
| **Linear** | 项目管理 | 深色+紫色 | 界面模拟演示、AI agent 工作流展示 |
| **Superhuman** | 邮件客户端 | 浅色极简 | 场景化模拟交互、价值量化对比 |
| **Notion** | 知识工作区 | 浅色极简 | 视频演示为主 CTA、信任背书数据 |
| **Superwhisper** | macOS 语音输入 | 深色+蓝紫渐变 | 动态字母矩阵、代码演示、"Just speak" |

## 核心经验总结

### 1. 文案 — "一句话说清价值"

**规律**：顶级产品的主标语都在 **5 个词以内**，直击用户痛点。

| 产品 | 主标语 | 字数 |
|------|--------|------|
| Raycast | "Your shortcut to everything." | 4 词 |
| Linear | "The product development system" | 4 词核心 |
| Superwhisper | "Just speak. Write faster." | 4 词 |
| Notion | "The AI workspace that works for you." | 7 词 |

**经验提炼**：
- 主标语用 **动词驱动**（speak, write, shortcut），给人行动感
- 副标语补充 **具体场景和机制**（how it works）
- 情感化收尾文案升华价值（Raycast: "It's not about saving time. It's about feeling like you're never wasting it."）
- **避免技术术语**，用用户能感知的语言

**Speakin 改进方向**：
- 原："按住 Fn，让语音变成文字" — 描述了功能，但缺乏 punch
- 新思路：更短、更有力、动词驱动。强调"说"和"输入"的直觉联系

### 2. 宣传标语 — 层次递进结构

**规律**：优秀首页的文案有 **3 层递进**：

```
第 1 层：情感钩子（Hero 主标语）     → 吸引注意力
第 2 层：功能解释（Hero 副标语）     → 说清楚是什么
第 3 层：价值升华（Features 前的引导）→ 为什么要用
```

**示例（Superwhisper）**：
1. "Just speak. Write faster." — 情感钩子
2. "Turn your voice into polished text." — 功能解释
3. "Select an app, press ⌥+space and start dictating" — 操作引导

**Speakin 改进方向**：设计 3 层递进文案，避免一开始就解释全部功能

### 3. 动画 — "展示而非讲述"

**规律**：最有效的动画不是装饰性的，而是 **演示产品实际工作流程**。

| 产品 | 动画类型 | 效果 |
|------|---------|------|
| Raycast | 模拟键盘按压 → 弹出启动器 | 让用户"看到"操作 |
| Linear | 模拟任务从 Todo → In Progress | 展示工作流 |
| Superwhisper | 动态字母矩阵 + 代码编辑器演示 | 科技感 + 实际场景 |
| Superhuman | 模拟聊天 → AI 自动建议预订 | 展示 AI 能力 |

**关键原则**：
- **Hero 动画 = 产品 Demo**，不是装饰
- 打字机效果适合文字产品（✓ 我们已有，但需要更好的包装）
- 波形动画直觉传达"正在听"（✓ 我们已有）
- 需要增加：**完整的工作流演示**（按键 → 录音 → 文字出现）

**Speakin 改进方向**：
- 将 demo 区做成完整的 **3 阶段流动画**：① fn 按下 → ② 声波动画 → ③ 文字逐字出现
- 增加过渡流动感，让三个阶段自然衔接
- 加入微妙的背景粒子/光效（Superwhisper 风格）

### 4. 布局节奏 — "呼吸感"

**规律**：所有优秀首页都遵循 **松-紧-松** 的节奏。

```
[大留白 Hero]        ← 松：视觉聚焦，只有标语+CTA
[密集 Feature Grid]  ← 紧：信息密集，展示功能丰富性
[留白 社会证明]      ← 松：让用户消化
[紧凑 安装指南]      ← 紧：操作步骤
[留白 Footer]        ← 松：收尾
```

### 5. 配色 — 深色主题是主流

5/5 的产品使用深色背景（或至少提供深色模式），原因：
- 开发者/效率工具用户偏好深色
- 深色背景让 **渐变色和光效更出彩**
- 传达"专业、极客"的调性

**品牌色规律**：紫色/蓝紫渐变是生产力工具的主流选择（Raycast、Linear、Superwhisper 都用紫色系）

## 对 Speakin 主页的改进方案

### 文案重写
- **主标语**：简短有力，4-5 词，动词驱动
- **副标语**：一句话说清机制
- **特性标语**：每个 feature 用一个动词开头

### 动画升级
- Hero 区：完整的 3 阶段产品流演示（按键→录音→输出）
- 背景：缓慢流动的渐变光斑
- 功能卡片：悬浮时内部图标微动
- 整页：视差滚动效果

### 视觉升级
- 更大胆的渐变色（不只是紫色，加入蓝紫→粉紫过渡）
- 毛玻璃质感卡片
- 更大的字号和更多留白
- 增加信任背书区域（技术栈 badge）
